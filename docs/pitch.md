# CLOUD DREAMERS（云端筑梦师）Demo 文案

## 创作故事：第一张支票

1903 年冬天，基蒂霍克的沙丘像一张巨大的蓝图纸，风把海盐与细沙吹进每一道铅笔线里。你坐在办公室的煤油灯下，翻看两位来自自行车工坊的兄弟寄来的信：他们没有华丽的头衔，只有一摞被砂砾磨毛的试验记录、一段段关于翼型数据的争论、以及一个看似荒唐却又极其具体的请求——“请给我们 500 美元，买云杉木、拉线，以及一台能持续运转的 12 马力发动机。”你并不缺少投机项目，但你从未见过如此像工程师的求助：每一分钱都被写成了零件清单，每一次失败都被写成了可复现的原因分析。

你签下支票的一瞬间，事情就不再是“他们的梦想”。那是一份契约：他们负责把空气变成可测量、可计算的对象，而你负责把试飞变成可以交付的产品。第一架飞机诞生后，军方与媒体闻讯而来，订单像潮水一样涌进来：有人要能在海风里保持航向，有人要能带着相机在战场上空稳定盘旋，有人要把炸弹送到后方工厂的屋顶。你开始明白，航空史并不是一条直线，而是一串互相牵制的选择：升力与阻力、可靠与极限、商业窗口与地缘压力。你要做的并不是“造出最强的飞机”，而是在每一个时代的约束里，造出“能被组织起来持续交付”的飞机。

在你的仓库里，机翼、发动机、机身像词汇一样被摆上货架；在你的科技树里，机型与关键模块像句法一样相互连接。你每一次提交订单，得到的不是一句“成功/失败”，而是一份带着时代背景的工程报告：为什么这套组合在这个年代能成立，为什么同样的组合到了下一代会被新的材料、气动与运营模型击败。你投下的不是一张支票，而是一条跨越百年的技术与商业路径。

## 核心亮点：机型科技树 + 模块研发 + 百科式档案（玩着学）

本 Demo 的核心亮点是把“航空史知识图谱”做成可玩的主循环：玩家不是背参数，而是在订单压力下做工程取舍。科技树以真实机型为节点，每个机型拆成机翼/发动机/机身三大模块；关键模块点亮后会解锁后继机型或分支路线，形成类似“坦克世界”式的机型演化网络。每个机型与模块都配有分栏百科档案（材料学、空气动力学、商业、地缘政治、性能解读），玩家在解锁与装配时自然接触到“为什么这样设计”“这项技术如何扩散”“它为何赢/为何输”。提交订单后弹出长篇工程报告，解释指标差距、给出改进建议，并把你的组合与真实历史原型关联起来，完成“操作→反馈→知识沉淀”的闭环。

## 使用场景：黑客松评审/科普展台的一体化 Demo

这个 Demo 特别适合用在“5 分钟可讲清、10 分钟可上手、15 分钟能看到成长”的展示场景：黑客松路演、科普馆展台、或高校课程中的互动演示。评审或观众打开页面后先经历“莱特兄弟求投资”的叙事引导，立刻进入订单驱动的玩法：从仓库选择模块组装，点击提交得到带时代背景的工程反馈，接着用奖励推进科技树解锁新机型与新模块。展台讲解员可以用“同一订单的多种解法”来演示知识点：为什么单翼更快但更挑结构、为什么任务机要牺牲极速换滞空、为什么喷气时代的瓶颈从“能飞”转向“可持续运营”。整个过程不依赖复杂物理模拟，但仍能呈现航空史的复杂性与连续迭代感，适合快速落地、便于扩展内容。
